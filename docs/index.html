Quick start

Simple things stay simple

Plain text is a one-liner. Formatted content uses a builder callback. TypeScript autocomplete guides you.

import { OdtDocument } from "odf-kit";

const doc = new OdtDocument();
doc.setMetadata({ title: "Quarterly Report" });
doc.setFooter("Page ###");

doc.addHeading("Q4 Results", 1);
doc.addParagraph("Revenue exceeded expectations across all regions.");

doc.addTable([
  ["Region",  "Revenue",   "Growth"],
  ["North",   "$2.1M",     "+12%"],
  ["South",   "$1.8M",     "+8%"],
], { border: "0.5pt solid #000" });

doc.addParagraph((p) => {
  p.addText("Status: ");
  p.addText("Approved", { bold: true, color: "green" });
});

const bytes = await doc.save();
// ‚Üí Uint8Array ‚Äî valid .odt, opens in LibreOffice
Why odf-kit

Built to fill a real gap

The JavaScript ecosystem had no maintained library for ODF document generation. odf-kit changes that.

TypeScript-first

Full type definitions ship with the package. Your editor shows every method, every option, every valid value ‚Äî as you type.

Single dependency

Only jszip at runtime. No XML parsers, no heavy frameworks. Lightweight and audit-friendly.

ODF 1.2+ compliant

Generates proper ZIP-packaged .odt files per ISO/IEC 26300. Validated against LibreOffice 24.2.

Open standards matter

ODF is the ISO standard for documents. Used by governments, schools, and nonprofits worldwide who choose vendor independence.

Developer-friendly API

Simple things are one-liners. Complex output uses builder callbacks. Both bold: true and fontWeight: "bold" work.

No prior art

The only JS ODF library (simple-odf) was abandoned in 2021. It had no table support and only generated flat XML, not proper .odt files.
